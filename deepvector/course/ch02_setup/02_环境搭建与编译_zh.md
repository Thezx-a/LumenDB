## 2.1 克隆仓库

```bash
git clone --recursive https://github.com/Thezx-a/DeepVector.git
cd DeepVector
```

如果忘记 `--recursive`:

```bash
git submodule update --init --recursive
```

---

## 2.2 编译 C++ 服务器

```bash
cmake -B build -G Ninja \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_CXX_COMPILER=g++-12 \
    -DUSE_AVX2=ON

cmake --build build -j$(nproc) --target deepvector_server
```

> **逐行解释**：
> - `-B build`：指定构建目录
> - `-G Ninja`：使用 Ninja (比 Make 快)
> - `-DUSE_AVX2=ON`：启用 AVX2 SIMD 加速
> - `-j$(nproc)`：并行编译，使用所有 CPU 核心

验证：

```bash
./build/server/deepvector_server --help
```

预期输出：显示 `--port`, `--dim`, `--data-dir` 等参数。

## 2.3 安装 Python 依赖

```bash
# 最小安装
pip install httpx pydantic sentence-transformers

# 完整安装 (推荐)
pip install fastapi uvicorn mcp pytest pytest-asyncio
```

验证：

```bash
python -c "from agent.config import load_config; print(load_config())"
```

---

## 2.4 配置 Ollama (可选)

```bash
# 安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 拉取模型
ollama pull nomic-embed-text   # 嵌入模型 (~274MB)
ollama pull qwen2.5:3b         # 轻量 LLM (~1.7GB)
# 或
ollama pull qwen2.5:7b         # 更强 LLM (~4.7GB)

# 启动服务
ollama serve
```

---

## 2.5 启动验证

```bash
# 终端 1: 启动 DeepVector
./build/server/deepvector_server --port 8080 --dim 384

# 终端 2: 健康检查
curl http://localhost:8080/health
# {"status":"ok","vectors":0,"dim":384}

# 终端 3: 启动 Agent
python -m agent.server.app
```
